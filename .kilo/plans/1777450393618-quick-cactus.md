# Plan : Création du module wrapper-llama

## Objectif
Créer un package Python publiable sur PyPI, utilisant `uv` et `pyproject.toml`, avec une structure `src/` standard. Le module encapsule `llama-server` (llama.cpp) pour un pilotage facile depuis Python.

## Structure de fichiers à créer

```
D:\GIT\wrapper-llama\
├── pyproject.toml                    # Config uv/setuptools, métadonnées PyPI
├── .env.example                      # Variables d'environnement exemple
├── .gitignore                        # Ignore __pycache__, .env, dist/, etc.
├── README.md                         # (optionnel - ne sera pas créé sauf si demandé)
├── src/
│   └── wrapper_llama/
│       ├── __init__.py               # Export public de LlamaServerManager
│       └── server.py                 # Classe LlamaServerManager (depuis PROJET.md)
└── examples/
    └── basic_chat.py                 # Script exemple avec .env
```

## Étapes d'implémentation

### 1. Créer `pyproject.toml`
- Nom du package : `wrapper-llama`
- Version initiale : `0.1.0`
- Auteur/description basiques
- Dépendance : `python-dotenv` (pour le support .env dans l'exemple)
- Build system : `hatchling` (compatible uv, standard moderne)
- Structure `src/` avec `[tool.hatch.build.targets.wheel] packages = ["src/wrapper_llama"]`

### 2. Créer `src/wrapper_llama/server.py`
- Reprendre la classe `LlamaServerManager` du PROJET.md à l'identique
- Aucune modification fonctionnelle

### 3. Créer `src/wrapper_llama/__init__.py`
- Import public : `from .server import LlamaServerManager`
- Définition de `__all__` et `__version__`

### 4. Créer `examples/basic_chat.py`
- Script utilisant `python-dotenv` pour charger `.env`
- Utilise `LlamaServerManager` en context manager
- Envoie une requête `/v1/chat/completions`
- Affiche la réponse

### 5. Créer `.env.example`
- Variables : `LLAMA_EXE_PATH`, `LLAMA_MODEL_PATH`, `LLAMA_GPU_LAYERS`

### 6. Créer `.gitignore`
- Ignore : `__pycache__/`, `.env`, `dist/`, `build/`, `*.egg-info/`, `.venv/`

### 7. Initialiser le projet avec uv
- `uv sync` pour créer le venv et installer les dépendances

## Aucune modification du code existant
La classe `LlamaServerManager` est reprise telle quelle de PROJET.md.
